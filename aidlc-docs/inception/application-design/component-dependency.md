## Dependency Rules

- The **CLI layer** has no knowledge of crypto or I/O implementation details
- The **Service layer** has no knowledge of the CLI layer (enabling future GUI/web/mobile frontends)
- The **Crypto layer** components (`AesGcmEngine`, `KeyManager`) do not depend on each other
- The **I/O layer** components (`FileFormatHandler`, `StreamProcessor`) do not depend on the Crypto layer
- Dependencies flow strictly downward: CLI â†’ Service â†’ Crypto/IO

## Data Flow â€” Encryption

```
User
  |  args: [sourcefile] [password]
  v
ArgumentParser
  |  ParsedArgs(ENCRYPT, sourcePath, password)
  v
Main
  |  sourcePath, password
  v
EncryptionService
  |---> KeyManager          generates salt, IV; derives or generates key
  |---> FileFormatHandler   builds LcryptHeader
  |---> StreamProcessor     opens source and destination streams
  |       |---> FileFormatHandler   writes header bytes to output stream
  |       |---> AesGcmEngine        streams ciphertext + auth tag to output stream
  |
  |  EncryptionResult(outputPath, autoGeneratedKey?)
  v
Main
  |  success message, optional key warning
  v
ConsoleIO (stdout)
```

## Data Flow â€” Decryption

```
User
  |  args: [file.lcrypt] [password?]
  v
ArgumentParser
  |  if no password: ConsoleIO.readPasswordInteractively()
  v
Main
  |  sourcePath, password
  v
DecryptionService
  |---> StreamProcessor     opens source stream
  |       |---> FileFormatHandler   reads and validates LcryptHeader
  |---> KeyManager          derives key from password + header.salt + header.argon2Params
  |---> StreamProcessor     opens destination stream
  |       |---> AesGcmEngine        decrypts stream; verifies GCM auth tag
  |
  |  DecryptionResult(outputPath)  OR  ServiceException (wrong password / tampered)
  v
Main
  |  success message  OR  plain-language error
  v
ConsoleIO (stdout / stderr)
```

## External Dependencies

| Library | Version Strategy | Purpose |
|---|---|---|
| Bouncy Castle (`bcprov-jdk18on`) | Exact version pinned in `build.gradle` | Argon2id key derivation |
| Java 21 JDK standard library | Bundled with JDK | AES-GCM (`javax.crypto`), `SecureRandom`, file I/O, `System.console()` |

No other third-party runtime dependencies. Bouncy Castle is the single external library.

---

## Communication Patterns

All communication between components uses direct Java method calls within a single process. There are no inter-process or network communication patterns in Phase 1. Components share data through value objects (`ParsedArgs`, `LcryptHeader`, `Argon2Params`, `EncryptionResult`, `DecryptionResult`) rather than shared mutable state.
