# LlamaCrypt — Unit of Work Story Map

Unit 0 is infrastructure — it has no user stories. All five user stories are
delivered across Units 1–3, with Unit 3 being the first unit where any story
can be fully exercised end-to-end from the command line.

---

## Story-to-Unit Mapping

| Story | Title | Primary Unit | Supporting Units |
|---|---|---|---|
| — | Repository & CI/CD Setup | Unit 0 (infrastructure) | — |
| US-01 | Encrypt a file with a password | Unit 1 + Unit 2 | Unit 3 |
| US-02 | Encrypt a file without a password | Unit 1 + Unit 2 | Unit 3 |
| US-03 | Auto-key generation warning | Unit 3 | Unit 2 (returns key string) |
| US-04 | Decrypt a `.lcrypt` file | Unit 1 + Unit 2 | Unit 3 |
| US-05 | Receive clear feedback on operation outcome | Unit 3 | Unit 2 (ServiceException messages) |

---

## Detailed Story Breakdown by Unit

### US-01 — Encrypt a File with a Password

| Acceptance Criterion | Unit | Component |
|---|---|---|
| AC-1: Output `.lcrypt` file created | Unit 1 + Unit 2 | Core encrypt pipeline |
| AC-2: Source file preserved | Unit 2 (EncryptionService) | Never modifies source |
| AC-3: Success feedback with output path | Unit 3 (ConsoleIO) | `printSuccess()` |
| AC-4: Missing file — error feedback | Unit 3 (ArgumentParser) | `validate()` |
| AC-5: Unreadable file — error feedback | Unit 1 (StreamProcessor) + Unit 3 (ConsoleIO) | `ProcessingException` → `printError()` |

---

### US-02 — Encrypt a File Without a Password

| Acceptance Criterion | Unit | Component |
|---|---|---|
| AC-1: Output `.lcrypt` file created | Unit 1 + Unit 2 | Core encrypt pipeline (random key path) |
| AC-2: Auto-generated key displayed | Unit 2 (EncryptionResult.autoGeneratedKey) + Unit 3 (ConsoleIO) | `printAutoKeyWarning()` |
| AC-3: Source file preserved | Unit 2 (EncryptionService) | Never modifies source |
| AC-4: Success feedback with output path | Unit 3 (ConsoleIO) | `printSuccess()` |
| AC-5: Missing file — error feedback | Unit 3 (ArgumentParser) | `validate()` |

---

### US-03 — Auto-Key Generation Warning

| Acceptance Criterion | Unit | Component |
|---|---|---|
| AC-1: Warning is prominent (⚠ WARNING: prefix) | Unit 3 (ConsoleIO) | `printAutoKeyWarning()` |
| AC-2: Warning content is clear | Unit 3 (ConsoleIO) | Warning message wording |
| AC-3: Warning appears before success message | Unit 3 (Main + ConsoleIO) | Output ordering in Main |
| AC-4: Key is on its own labelled line | Unit 3 (ConsoleIO) | Key display format |

---

### US-04 — Decrypt a `.lcrypt` File

| Acceptance Criterion | Unit | Component |
|---|---|---|
| AC-1: File restored with original filename | Unit 1 + Unit 2 | Core decrypt pipeline |
| AC-2: Interactive prompt hides input | Unit 3 (ConsoleIO) | `readPasswordInteractively()` |
| AC-3: Success feedback with output path | Unit 3 (ConsoleIO) | `printSuccess()` |
| AC-4: Wrong password — error feedback | Unit 1 (AesGcmEngine) + Unit 2 (ServiceException) + Unit 3 (ConsoleIO) | Auth tag failure path |
| AC-5: Corrupt/tampered file — error feedback | Unit 1 (AesGcmEngine) + Unit 2 (ServiceException) + Unit 3 (ConsoleIO) | GCM auth tag verification |

---

### US-05 — Receive Clear Feedback on Operation Outcome

| Acceptance Criterion | Unit | Component |
|---|---|---|
| AC-1: Plain-language success messages | Unit 3 (ConsoleIO) | `printSuccess()` |
| AC-2: Plain-language error messages (no internals) | Unit 2 (ServiceException) + Unit 3 (ConsoleIO) | `printError()` |
| AC-3: Non-zero exit code on failure | Unit 3 (ConsoleIO) | `exit(1)` |
| AC-4: Zero exit code on success | Unit 3 (ConsoleIO) | `exit(0)` |
