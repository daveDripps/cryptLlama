# LlamaCrypt — Services

Services sit between the CLI layer and the crypto/I/O components. They own the full
encrypt and decrypt workflows and have no dependency on the CLI layer, which enables
future GUI, web, and mobile frontends to reuse them without modification (NFR-08).

---

## Service 1 — EncryptionService

**Class**: `com.llamacrypt.service.EncryptionService`

### Purpose
Orchestrates the complete file encryption workflow from password/key decision through to the finished `.lcrypt` output file.

### Responsibilities
- Accept a source file path and an optional password (`char[]`, may be null)
- If no password provided: generate a random 256-bit key and the Base64-encoded key string for display (US-02, US-03)
- If a password is provided: generate a random salt, derive a 256-bit key using Argon2id (US-01)
- Generate a random 12-byte IV/nonce
- Build the `LcryptHeader` (magic, version, Argon2id params, salt, IV)
- Write the header to the output `.lcrypt` file
- Stream-encrypt the source file content using `AesGcmEngine`, appending ciphertext and GCM auth tag after the header
- Return an `EncryptionResult` (output path, and the auto-generated key string if applicable)
- Zero all sensitive byte/char arrays after use (NFR-04)

### Collaborators
| Component | Role |
|---|---|
| `KeyManager` | Generate salt, IV, derive or generate key |
| `AesGcmEngine` | Perform AES-256-GCM stream encryption |
| `FileFormatHandler` | Build and write the `.lcrypt` header |
| `StreamProcessor` | Open and manage buffered file streams |

### Method Signature
```java
public EncryptionResult encrypt(Path sourceFile, char[] password) throws ServiceException;
```

**Input**:
- `sourceFile` — path to the plaintext file to encrypt
- `password` — user-supplied password as `char[]`; `null` triggers auto-key generation

**Output**: `EncryptionResult` value object:
- `outputPath` — path to the newly created `.lcrypt` file
- `autoGeneratedKey` — Base64-encoded key string, or `null` if a password was provided

**Throws**: `ServiceException` wrapping any crypto, format, or I/O failure

### Workflow Sequence
```
EncryptionService.encrypt(sourceFile, password)
  1. Determine key strategy:
     a. password == null → KeyManager.generateRandomKey() + KeyManager.generateSalt()
     b. password provided → KeyManager.generateSalt() → KeyManager.deriveKey(password, salt, params)
  2. KeyManager.generateIv()
  3. FileFormatHandler.buildHeader(salt, iv, argon2Params)
  4. Compute outputPath = sourceFile + ".lcrypt"
  5. StreamProcessor.process(sourceFile, outputPath, stream →
       FileFormatHandler.writeHeader(out, header)
       AesGcmEngine.encryptStream(in, out, key, iv)
     )
  6. KeyManager.clearSensitiveBytes(key)
  7. If password provided: KeyManager.clearSensitiveChars(password)
  8. Return EncryptionResult(outputPath, autoGeneratedKey)
```

## Service 2 — DecryptionService

**Class**: `com.llamacrypt.service.DecryptionService`

### Purpose
Orchestrates the complete file decryption workflow from reading the `.lcrypt` header through to restoring the original plaintext file.

### Responsibilities
- Accept a source `.lcrypt` file path and a password (`char[]`, may be null — interactive prompt handled upstream by ConsoleIO)
- Read and validate the `.lcrypt` file header using `FileFormatHandler`
- Extract salt, IV, and Argon2id parameters from the header
- Derive the 256-bit AES key from the password and header-stored salt using Argon2id
- Stream-decrypt the file body using `AesGcmEngine`, which verifies the GCM authentication tag
- On authentication tag failure: raise `ServiceException` with a plain-language message ("Incorrect password or the file is corrupted")
- On success: write the decrypted content to a file with the `.lcrypt` extension removed
- Return a `DecryptionResult` (output path)
- Zero all sensitive byte/char arrays after use (NFR-04)

### Collaborators
| Component | Role |
|---|---|
| `FileFormatHandler` | Read and parse the `.lcrypt` header |
| `KeyManager` | Derive key from password + salt from header |
| `AesGcmEngine` | Perform AES-256-GCM stream decryption + auth tag verification |
| `StreamProcessor` | Open and manage buffered file streams |

### Method Signature
```java
public DecryptionResult decrypt(Path sourceFile, char[] password) throws ServiceException;
```

**Input**:
- `sourceFile` — path to the `.lcrypt` file to decrypt
- `password` — user-supplied or interactively collected password as `char[]`

**Output**: `DecryptionResult` value object:
- `outputPath` — path to the restored plaintext file

**Throws**: `ServiceException` wrapping format errors, auth tag failures, or I/O failures

### Workflow Sequence
```
DecryptionService.decrypt(sourceFile, password)
  1. StreamProcessor opens sourceFile for reading
  2. FileFormatHandler.readHeader(in) → LcryptHeader (validates magic, version)
  3. KeyManager.deriveKey(password, header.salt, header.argon2Params)
  4. Compute outputPath = sourceFile with ".lcrypt" removed
  5. AesGcmEngine.decryptStream(in, out, key, header.iv)
     → On AEADBadTagException: throw ServiceException("Incorrect password or the file is corrupted")
  6. KeyManager.clearSensitiveBytes(key)
  7. KeyManager.clearSensitiveChars(password)
  8. Return DecryptionResult(outputPath)
```
